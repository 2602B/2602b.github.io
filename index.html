Introduction To VEX Programming

Robotics programming can be daunting to new members, but it does not need to be.

This website is intended to be used as a guide and not a source of software. When you get a idea, try it!

You'll learn far more by testing the limits of your own code than you will from this website.

Resources

There are many resources available to you. The most important in C++ is the VEX API.

VEX Forum is a great place to ask questions about anything VEX related.

I recommend Harvard CS50 for learning more about programming. The Scratch, Python, and C lectures are particularly useful.

What To Program With

There are a variety of ways to program using the VEXCode software suite.

IDE Details Languages
VEXCode Online This is the easiest and quickest to use, only requiring a web connection and light programming knowledge. Blocks, Python, C++
VEXCode Pro An offline editor for C++ programming with built in help menus like the online version. C++
VEXCode VSC A Visual Studio Code extension for C++ and Python programming. C++, Python
Purdue Pros An independent C++ editor to VEXCode. This framework uses different syntax. C++

Since Purdue Pros uses different syntax, it will not be covered in this guide.

Getting Started

Open up your editor of choice, and create a blank project.

It's very important the project name is clear on what it does, otherwise, you may run the wrong program in competition.

But for now, call it testUserControl.

You should now see block, Python or C++ code in front of you.

If you are using C++, the most important file is src > main.cpp

Setting Up Motors

Whenever possible, I recommend setting up motors in the Devices tab of the VEXCode.

  1. 1. In the Motors section, click the "+" button to add a new motor.
  2. 2. In the Name field, enter a descriptive name for the motor (e.g. "LeftMotor", "RightMotor", "ArmMotor").
  3. 3. In the Device field, select the port that the motor is plugged into.
  4. 4. Click the Add button to save the motor.
  5. 5. Repeat the above steps for each motor you want to add.

Setting Up Controllers

Similar to setting up motors, you can set up controllers in the Robot tab of VEXCode.

  1. 1. In the Controllers section, click the "+" button to add a new controller.
  2. 2. In the Name field, enter a descriptive name for the controller (e.g. "Controller1", "Controller2").
  3. 3. In the Type field, select the type of controller you are using (e.g. VEX Gamepad, VEX Wireless Gamepad).
  4. 4. Click the "Add" button to save the controller.

Putting It All Together

In order to drive the robot, there first must be code bridging the gap between controller and hardware.

User Control code boils down to this:

  1. 1. Initialize
  2. 2. Get joystick information
  3. 3. Respond to joystick information
  4. 4. Respond to any buttons pushed
  5. 5. Repeat #2

Competition Template

The competition template is a good place to start.

When the competition organizers begin the autonomous and driver control period, the code must be organized to know which function to run.

The importance of this template cannot be overlooked, which is why I highly recommend testing your code with a competition switch.
